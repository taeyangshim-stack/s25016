Subject[S25016] 원점관리 UI 검토 요청 리마인드 및 원점 로깅 DB 구성 안내

안녕하십니까, 에스피시스템스 심태양입니다.

두 가지 건으로 안내 드립니다.

1. 원점관리 및 수동조깅 UI 개선 - 검토 요청 (리마인드)

어제(2/11) 메일로 안내드린 원점관리 UI 개선 및 수동조깅 UI 개선 건에 대해 검토 요청을 드린 바 있습니다.

해당 내용은 2/10(월) 기능 컨펌 완료된 아래 4건에 대한 UI 보완 배포 사항입니다.

컨펌 완료 항목 (4건)
• PL-2025-152 : LDS 센서 기반 원점 인식 (이마룬)
• PL-2025-155 : 홀 센서 기반 원점 인식 (최광년)
• PL-2025-153 : 십자레이저 갠트리 원점 오차 확인 (최광년)
• PL-2026-170 : 갠트리 상대이동 조깅 기능 (박주한)

첨부 동영상(원점확인 동작, A/B라인 수동조깅 동작)과 함께 확인 후 승인 회신 부탁드립니다.
현재 용접 이슈로 인해 2/3부로 운영SW가 롤백된 상태이므로 현장 즉시 배포는 어려운 상황이나, 승인 완료 시 배포 준비를 완료해 두어 용접 이슈 해결 후 신속하게 적용할 수 있도록 하겠습니다.

2. 원점 로깅 기능 - 별도 DB 구성 안내

상기 원점관리 기능과 연계하여, 원점 측정 이력을 영구 보존하기 위한 로깅 기능을 구성하였습니다.

AS-IS (기존)

  • 홀센서/LDS 측정값 UI 출력만
  • 프로그램 종료 시 데이터 휘발
  • 원점 변화 추이 확인 불가
➔

TO-BE (개선)

  • 측정값 SQLite DB에 영구 저장
  • 원점 정도 변화 추이 확인 가능
  • CSV 내보내기로 이력 분석 활용

데이터 흐름:

RAPID센서 구동 & 측정
 ➔
운영SW (C#)오차 계산
 ➔
SQLite DB영구 저장
 ➔
CSV내보내기

구성 방식:

  • 별도 SQLite DB 파일 신규 생성: DbData\gantry_origin_record.sqlite
  • 홀센서/LDS 각각 별도 테이블로 분리 저장
  • 측정 프로세스 완료 시 자동으로 DB에 기록
  • CSV 내보내기 기능으로 이력 데이터 활용 가능

DB 테이블 구성:

홀센서 테이블 (hole_logging)
ColumnType설명
timestampTEXT측정 완료 시간 (Primary Key)
gantryIdNUMERIC갠트리 호기 ID
resultX1AxisNUMERICX1축 오차 (mm)
resultX2AxisNUMERICX2축 오차 (mm)
resultXSkewNUMERICX Skew 각도
measurementSuccessfulTEXT측정 성공 여부
LDS 테이블 (lds_logging)
ColumnType설명
timestampTEXT측정 완료 시간 (Primary Key)
gantryIdNUMERIC갠트리 호기 ID
resultXAxisNUMERICX축 오차
resultYAxisNUMERICY축 오차
resultZAxisNUMERICZ축 오차
measurementSuccessfulTEXT측정 성공 여부

원점 체크 팝업 UI (A/B라인 공통):

홀센서(X1, X2, Skew) / LDS(X, Y, Z) 측정 및 기록 저장 / 수동 원점 체크

기대 효과:

  • 원점 정도 변화 추이 확인 → 설비 상태 수렴/발산 여부 판단
  • 장비 유지보수 시 과거 이력 기반 의사결정 지원
  • 호기별 데이터 비교 분석 가능

본 기능은 디파인시스템 박주한 주임이 개발 완료하였으며, RobotStudio 시뮬레이션 테스트 후 현장 실물 테스트를 진행할 예정입니다.

상기 내용 검토 후 회신 부탁드리겠습니다.
