이동혁 프로님께,

안녕하십니까, 에스피시스템스 심태양입니다.

PL-2025-125로 등록된 '용접기 용접 모드 상실' 이슈와 관련하여, 문제의 원인을 파악하고 재현하기 위한 구체적인 점검 계획을 수립하여 공유드립니다.

1. 점검 목적

용접 모드 5번 소실 및 User Weld Set 변경 현상(`PL-2025-125`)이 발생하는 원인을 체계적으로 분석하고, 재현 조건을 명확히 파악하는 것이 목표입니다. 특히, 12월 16일에 발생했던 용접 설정 변경 문제 상황을 재현하는 데 중점을 둡니다.

2. 점검 방법 (상세 테스트 시나리오)

랜선 연결 상태 변경과 장비(로봇/용접기) 재부팅을 조합하여 문제 발생의 원인(통신 문제, 장비 자체 문제 등)을 분리하여 확인합니다. 각 시나리오 전에는 반드시 용접기 설정이 정상 상태(용접 모드 5번 존재, User Weld Set 활성)인지 확인합니다.

순서 테스트 시나리오 세부 조치 내용 점검 목적 및 예상 결과
1 랜선을 뽑았다가 재연결
  1. 로봇과 용접기 연결 랜선을 뽑습니다.
  2. 각각 1초, 5초, 10초 대기 후 랜선을 다시 연결합니다.
  3. 각 재연결 후 즉시, 그리고 1분 경과 후 용접기 설정값(모드 5, User Weld Set)을 확인합니다.
 통신 단절 및 재연결 과정에서 설정 변경이 발생하는지 확인합니다. 짧은/긴 통신 단절이 미치는 영향을 파악하여 통신 복구 로직 오류 여부를 추정합니다.
2 랜선 분리 후 로봇 재부팅
  1. 로봇과 용접기 연결 랜선을 뽑습니다.
  2. 로봇만 전원을 껐다가 다시 고 완전히 부팅될 때까지 기다립니다.
  3. 용접기 설정값(모드 5, User Weld Set)을 확인합니다. (정상이어야 함)
  4. 랜선을 다시 연결하고, 즉시 그리고 1분 경과 후 용접기 설정값을 확인합니다.
 로봇 재부팅 시 설정값이 변동되는지, 또는 재부팅 후 랜선 재연결 시 로봇이 잘못된 값을 용접기에 전송하는지 확인합니다.
3 랜선 분리 후 용접기 재부팅
  1. 로봇과 용접기 연결 랜선을 뽑습니다.
  2. 용접기만 전원을 껐다가 다시 고 완전히 부팅될 때까지 기다립니다.
  3. 로봇 측에서 용접기 설정값을 확인합니다. (정상이어야 함)
  4. 랜선을 다시 연결하고, 즉시 그리고 1분 경과 후 용접기 설정값을 확인합니다.
 용접기 자체 재부팅 시 설정값이 변동되는지, 또는 로봇과의 통신 없이 용접기 자체의 안정성(설정값 유지)을 점검합니다.
4 랜선 분리 후 용접기 & 로봇 재부팅
  1. 로봇과 용접기 연결 랜선을 뽑습니다.
  2. 로봇과 용접기 모두 전원을 껐다가 다시 고 완전히 부팅될 때까지 기다립니다.
  3. 랜선을 다시 연결하고, 즉시 그리고 1분 경과 후 용접기 설정값을 확인합니다.
 모든 장비가 초기 상태에서 시작하여, 통신 연결이 확립되는 과정에서 동기화 문제나 설정 교환 오류가 발생하는지 확인합니다.
5 랜선 10초 분리 후 재연결, 전체 시스템 재부팅
  1. 로봇과 용접기 연결 랜선을 뽑습니다.
  2. 10초간 대기 후 랜선을 다시 연결합니다.
  3. 로봇과 용접기 모두 전원을 껐다가 다시 고 완전히 부팅될 때까지 기다립니다.
  4. 완전히 부팅된 후 즉시, 그리고 1분 경과 후 용접기 설정값을 확인합니다.
 통신 단절 및 전체 시스템 재시작이 결합된 복합 시나리오에서 설정값이 변동되는지 확인하여 특정 조건에서만 발생하는 심각한 동기화 문제를 파악합니다.

3. 예상되는 다음 단계

성공적으로 문제 상황이 재현될 경우, 해당 시나리오를 통해 유저 설정값 변경 여부와 모드 소실 여부를 명확히 확인하여 문제의 재현 조건을 확립하고, 이를 바탕으로 원인 분석 방향을 설정할 계획입니다.

📌 중요 참고 사항

  • 각 테스트 시나리오 시작 전에는 반드시 용접기의 현재 설정값(특히 '용접 모드 5번'과 'User Weld Set')을 백업해 두시고, 정상 상태임을 확인해 주십시오.
  • 각 조치 후 용접기 설정값의 변동 여부를 정확하게 기록해야 합니다.
