배경 및 연동 구조

2대의 ABB IRB 1200-5/0.9 로봇이 4축 갠트리(X, Y, Z, R) 위에서 동시 용접을 수행하는 시스템입니다. 상위 UI는 ABB PC SDK를 통해 컨트롤러의 RAPID 변수에 직접 접근하여 실시간 모니터링과 제어를 수행합니다.

03

근본 원인 — PlanA vs PlanB 태스크 구조 차이

3.1 Mechanical Unit Group 구성 차이

항목 PlanA (RW 6.16.0025) PlanB (RW 6.16.1028)
rob1 그룹ROB_1만 (Motion Planner 1)ROB_1 + GantryRob + ELM_X (갠트리 통합)
rob2 그룹ROB_2 (Motion Planner 2)ROB_2 (Motion Planner 2)
Gantry 그룹GantryRob_7 + ELM_X (MP 3)없음 — rob1에 통합
핵심 차이
PlanA는 갠트리가 독립 그룹이므로 T_Gantry 태스크에서 별도 제어. PlanB는 갠트리가 Robot1 외부축에 통합되어 T_ROB1에서 직접 제어. 이로 인해 PlanB는 T_Gantry 태스크가 불필요해져 전체 태스크를 10개 → 8개로 축소.

주의사항 및 후속 작업
즉시 확인 필요

EIO.cfg I/O 신호 정의 확인 — Static.mod가 참조하는 모든 I/O 신호가 PlanB의 EIO.cfg에 정의되어 있어야 합니다.

입력: siLn1Current, siLn1Voltage, siLn1TouchActive, siLn1Arc_Est, siLn2Current, siLn2Voltage, siLn2TouchActive, siLn2Arc_Est, di17_Head1Shock, di19_Head2Shock, co001_AutoMode, co002_PrgRun, co004_MotorOn, pi53_SetTeachingPoint

출력: cgo05_AxisX_Current, intMoveHome_Head/RBT1/RBT2/Gantry, intTouch_1/2, intReHoldGantry_1/2, so_MoveG_PosHold, soLn1/2TouchActive, po03_Home, po08_RobotMoving, po53_SetTeachingPoint

공유 PERS 변수 초기값
tWeld1/2 (용접 툴데이터), wobjWeldLine1/2, wobjRotCtr1/2 (작업좌표계)가 T_ROB1/T_ROB2에서 올바르게 정의되어 있어야 T_BG에서 참조 가능.
