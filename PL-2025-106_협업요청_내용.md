## PL-2025-106 협업 요청: B라인 데이터 로딩 지연(0.4초) 문제 진단

**요청 사항:**

B라인에서 SDK를 통한 데이터 로딩 시 약 0.4초의 지연이 지속적으로 발생하고 있습니다. A라인과 데이터 크기는 유사한 것을 확인하였으며, 이에 네트워크 또는 컨트롤러단의 문제로 추정됩니다. 정확한 원인 파악을 위해 다음 협업을 요청드립니다.

1.  **네트워크 지연 시간 분석**
    *   **내용:** UI PC에서 각 라인 컨트롤러까지의 네트워크 응답 속도 차이를 분석합니다.
    *   **요청:** A라인 및 B라인 로봇 컨트롤러의 **IP 주소를 공유**해 주시면, 저희 측에서 `ping` 테스트를 통해 응답 시간을 측정하고 결과를 공유드리겠습니다.

2.  **컨트롤러 부하 상태 확인 지원**
    *   **내용:** B라인 컨트롤러의 높은 부하가 응답 지연의 원인일 수 있는지 확인합니다.
    *   **요청:** B라인 컨트롤러의 **CPU 사용률 등 부하 상태를 확인할 수 있도록 지원** 또는 가이드를 부탁드립니다. A라인과 비교 데이터가 필요합니다.

3.  **API 호출 테스트 협조**
    *   **내용:** 전체 UI의 복잡성을 배제하고, 순수하게 데이터 요청 API 자체의 성능을 비교합니다.
    *   **요청:** 저희가 A/B 라인에 동일한 API 호출을 하는 **최소 기능 테스트 스크립트를 작성하여 테스트를 진행**하고자 합니다. 관련하여 기술 지원이 필요할 시 협조 부탁드립니다.

위 진단 결과를 바탕으로 후속 조치를 논의하고자 합니다.
