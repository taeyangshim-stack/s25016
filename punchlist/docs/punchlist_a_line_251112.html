세부 펀치리스트

로직1. 알람 다수 발생 시 UI 프리징 해소

요청: SHI 담당: 박주한 상태: 11/12 배포

AS-IS

  • 펜던트 알람 처리 스레드가 명시적으로 구분되지 않아 UI 스레드와 혼선.
  • 다수 Warning 수신 시 UI 프리징 위험 존재.

TO-BE

  • 펜던트 알람 처리 스레드를 UI와 분리해 혼선 방지.
  • 동시 경보 발생 시에도 UI 프리징 없이 로그 확인 가능.

UI2. 속도 파라미터 설정

요청: SHI 담당: 박주한 상태: 11/12 배포

AS-IS

  • 실속도 모니터링 기능이 없어 로봇/갠트리 실제 속도를 확인 불가.
  • 각 속도 파라미터를 UI에서 제어하기 어려움.

TO-BE

  • 실속도 모니터링 화면 도입.
  • Speeddata 4개 필드를 모두 열람/셋업 가능하도록 변경.

UI3. DB 데이터 저장 오류 현상 개선

요청: SHI 담당: 박주한

AS-IS

  • 설정뷰 매크로 편집 탭에서 데이터 가져오기/저장 시 오류 발생.
  • 백업 파일명이 xxxx.hhmm.backup 형식이라 동일 분 단위 저장 시 중복으로 실패.

TO-BE

  • 데이터 가져오기/저장 시 백업 파일명을 hhmmss 단위로 생성.
  • 동일 분 내 반복 저장에도 파일 충돌 없이 백업 유지.

UI4. 엑셀 데이터 제어

요청: SHI 담당: 박주한

AS-IS

  • 엑셀 불러오기 시 데이터 삭제 여부만 확인하고 실제 업로드 여부와 무관하게 토치 모션/심데이터 초기화.
  • 작업자가 특정 경로의 파일을 기억해야 하는 번거로움.

TO-BE

  • 실제 파일 업로드 여부에 따라 토치 모션과 심데이터를 초기화.
  • 기준값 호출 버튼을 신설해 지정 경로의 엑셀을 즉시 열람/호출.

UI5. 심데이터 추가 칼럼 구성

요청: SHI 담당: 박주한

AS-IS

  • Ign_Move_Delay, ScrapeStart 값을 UI에서 관리할 방법이 없음.
  • 로봇 전달용 추가 데이터 필요.

TO-BE

  • 해당 칼럼을 심데이터 테이블에 추가하고 내부 DB 및 RAPID로 전달.
  • 이물질 제거/피딩 지연 등 세부 동작 제어 가능.

UI6. 심데이터 선후행 구분 추가 (A라인)

요청: SHI 담당: 박주한

AS-IS

  • 토치 모션에는 선/후행 구분이 있으나 심데이터에는 구분 정보가 없음.
  • 용접선 선후행을 별도 조건으로 인지하기 어려움.

TO-BE

  • 심데이터에 선/후행 플래그를 추가해 상황별 데이터를 분리 관리.
  • A라인 한정 요구사항으로 UI/내부 DB 반영.

UI7. 편의성 및 가시성 개선

요청: SHI 담당: 박주한

AS-IS

  • 체크박스가 작아 정확한 클릭이 어려움.
  • 테이블 칼럼명이 셀보다 작아 글자가 잘리는 문제.

TO-BE

  • 셀 선택만으로 체크박스 선택/해제가 가능하도록 변경.
  • 칼럼 폭, 글자 줄바꿈을 조정해 가시성 확보.
