2. 스마트랙(Smart Rack) 정의 + 대상 부재 분류

부재 유형별 스펙 포함

정의(운영 관점)

스마트랙은 소조립 공장 내 절단 소재의 입고·보관·공급 및 완성 소조블록의 적재·출하까지를 관리하는 공장 전체 물류 관리 시스템이며, AMR과 연계해 공장 내 자재 이동을 자동화하고 CAD/생산스케줄과 연동하여 적시적소(JIT) 자재 공급을 실현합니다.

입고 → 적치 → 피킹 → 공정공급 완성 → 적재 → 출하 형상/무게/무게중심 다양

대상 부재 분류 체계

스마트랙이 취급하는 부재를 4개 패밀리로 분류합니다. 상위 20% 유형이 물동량의 약 80%를 차지합니다.

패밀리 부재 유형 크기 범위 무게 형상 특성 물동량
판재류 외판, 내판, 격벽판, 갑판판 1.5~3.2m × 3~12m
두께 6~50mm		 0.2~3.0 ton 평판 / 곡판 ~40%
보강재류 T-bar, Flat bar, Angle bar, Bulb flat 2~12m × 0.1~0.4m
두께 8~25mm		 0.05~0.5 ton 선형(직선/곡선) ~30%
브라켓류 나비, 러그, 스티프너, 칼라플레이트 0.1~0.8m × 0.1~0.6m
두께 10~25mm		 0.01~0.1 ton 소형 다양 형상 ~15%
소조블록 평블록, 곡블록, 입체블록(완성품) 2~6m × 2~8m
높이 0.3~2m		 2~15 ton 3D 복합 구조 ~15%
유연 적치의 핵심 도전: 판재류+보강재류가 물동량 70%를 차지하지만, 소조블록(완성품)은 형상/CoG가 가장 복잡하고 무게가 가장 크므로 안전검증 난이도가 높습니다. → 1차년도는 판재·보강재 중심 MVP, 2차년도부터 소조블록 확장 권장.
왜 "구역 고정형 랙"이 아니라 "유연 적치(Free-Form)"여야 하나?

현실 제약

  • 소부재/소조블록은 형상·길이·폭·두께·무게중심(CoG)이 제각각
  • 공정 변동/우선순위 변경으로 재배치(reshuffling)가 빈번
  • 안전(전도/좌굴/국부변형) 요구가 높아 경험 의존이 큼

유연 적치가 주는 이득

  • 공간 효율: "칸"이 아니라 "지지점" 기반으로 고밀도 적치
  • 운영 유연성: 생산계획 변화 시 자동으로 적치 전략 변경
  • 안전 내재화: 센서+모델 기반 안정성 검증으로 표준화

3. 유연 적치 스마트랙 아이디어 (핵심)

"부재가 랙을 정의"
핵심 문장
고정 구획(슬롯) 기반이 아니라, 3D 형상 + CoG + 허용 변형/지지 조건에 따라 랙이 지지점(서포트)을 재구성하고, 시스템이 안전·접근성·공정연계성을 동시에 최적화합니다.

컨셉 A: "핀-그리드(다점 지지) + 빠른 수동 가변" (현실형 MVP)

구조는 단순하지만 성과가 빠릅니다. 자동화는 "완전 자동"이 아니라, 추천 지지점 자동 계산 + 가이드(작업자/크레인)로 시작합니다.

  • 랙 면에 촘촘한 홀 그리드 (예: 100~150mm 피치)
  • 서포트 핀/받침을 퀵 체인지로 위치 변경
  • 부재 스캔 → 시스템이 "권장 지지점"을 표시(LED/AR/라벨)
  • 장점: 저비용·고신뢰 / 단점: 완전 무인 어려움
TRL 빠르게 상승 인력 개입 일부

컨셉 B: "전동(Servo) 서포트 모듈 + 반자동 적치" (차별화 중간 단계)

서포트가 높이/각도를 자동으로 맞춰주고, 적치는 크레인/AMR이 수행(도킹/정렬은 센서로 보조)합니다.

  • 서포트 모듈: 전동 스크류잭/리니어액추에이터
  • 상단 패드: 회전(2축) + 미끄럼 방지 + 교체형(우레탄/강재)
  • 센서: 하중(로드셀), 접촉 감지, (선택) 변형 게이지
  • 장점: 자동화 체감 큼 / 단점: 모듈 단가, 유지보수 설계 중요
자동 정렬/수평화 모듈 신뢰성 관건

컨셉 C: "셀 기반 Self-Reconfiguring Rack" (세계 최초급 스토리)

랙 자체가 셀(Cell)로 구성되어, 부재가 들어오면 셀이 자동으로 올라오고(또는 내려가고), 안정 자세를 만든 후 적치합니다.

  • 다점 셀(예: 10×N) 각각이 높이 제어(서보/유압)
  • 부재 3D/CoG 기반으로 셀 높이/접촉점 최적화
  • 하중 분포 실시간 모니터링 → 위험 시 재분배
  • 장점: "형상 무관"의 정점 / 단점: 대형화·비용·안전 인증 난이도↑
특허/논문화 강함 개발 난이도 높음

아이디어를 "조선형"으로 만드는 차별화 포인트

  • 생산계획 연동 적치 전략: 3일 내 투입품은 접근성 최우선, 2주 대기품은 고밀도
  • 변형/품질 관점: 단순 보관이 아니라 허용 변형을 지키는 지지 설계
  • 야드/공장 두뇌: 크레인 동선·재배치 비용까지 포함한 "AI Yard Brain" 확장
추천 진행
1차년도에는 컨셉 A→B로 "성과/현장수용성"을 확보하고,
병렬로 컨셉 C의 핵심(셀 제어/안정성 검증/특허)을 PoC로 분리 개발하는 방식이 안전합니다.

모듈화 = 유지보수

모듈설명비고

빔/프레임 레일/슬라이딩 빔으로 지지점 간격 조절(옵션) 공간 효율↑

설계 원칙: 하드웨어는 단순·강건, 지능은 소프트웨어로 (현장 유지보수/확장성 관점)

서포트 상단 패드(예시)

회전(2축) 패드 미끄럼 방지(우레탄) 국부압력 분산(넓은 면)
  • 회전 패드: 부재의 미세 각도 차이를 흡수 → 편하중 감소
  • 스페이서/새들(Saddle): 곡면/모서리 부재 대응
  • 퀵 체인지: 패드 마모/손상 시 빠른 교체
  • 인덱싱: "장착 ID"를 시스템이 인식(정확한 지지조건 기록)

개념 스케치(SVG)

"칸"이 아니라 "지지점 배열"로 적치 자세를 만든다는 의미의 시각화입니다.

구조 안전(전도/좌굴/국부변형) 체크를 시스템에 내장하는 방법

안정성 체크(빠른 규칙 기반)

  • CoG 투영이 지지 다각형 내부인지(전도 위험)
  • 지지점 간격/배치가 최소 조건 만족인지
  • 로드셀 하중 분포가 허용 편차 이내인지

정밀 체크(모델 기반)

  • CAD 기반 단순화 모델로 국부압력 추정
  • 허용 변형/응력 기준(현장 표준)과 비교
  • 이상 시 지지점 재배치/높이 재조정

5. 인식/안전 알고리즘 (형상→지지점→검증)

SW가 차별화

데이터 파이프라인(권장)

  1. 입고 시 3D 스캔(또는 CAD 매칭) → 메쉬/포인트클라우드 생성
  2. 부재 무게/CoG 추정(실측 하중 + CAD 질량특성 결합)
  3. 적치 후보 자세 생성(회전/뒤집기 제한 포함)
  4. 지지점 후보 생성(접촉 가능한 면/리브/보강부 우선)
  5. 안전/변형/접근성 점수로 최적 조합 선택
  6. 실적치 후 로드셀/변형 센서로 검증·학습
포인트
초기에는 "완벽한 AI"보다 규칙+검증센서로 현장 신뢰를 만든 뒤, 데이터가 쌓이면 최적화를 강화하는 흐름이 빠릅니다.

최적화 목표(3종)

목표설명
안전성 전도/편하중/국부변형 최소화, 인터록 기반 자동 차단
접근성 향후 출고/공정투입 시 크레인/AMR 접근 경로 최소 이동
공정 연계성 생산스케줄 기반 우선순위 반영(JIT), 재배치 비용 포함
간단한 점수화 예시(설명용)

총점 = 0.50×안전 + 0.30×접근성 + 0.20×공정연계 (현장 합의로 가중치 조정)
안전 점수는 CoG/하중분포/국부압력의 패널티 합으로 계산.

6. AMR·크레인 인터페이스(도킹/통신/이재)

도킹/통신 규격

도킹(정렬) 방식 옵션

  • 패시브 마커: QR/AprilTag + 카메라(저비용)
  • LiDAR 기반: 반사판/형상 기준 정렬(분진·조명 대응)
  • 기계 가이드: 콘/레일로 최종 위치 결정(가장 강건)
조선 환경 고려
분진/오염/조명 변화가 큰 경우가 많아 "기계 가이드 + 센서 보조" 조합이 안정적입니다.

통신/제어(권장 골격)

  • 상위: WMS/WCS/입출고 S/W ↔ 스마트랙 제어(REST/MQTT 등)
  • 하위: 스마트랙 PLC ↔ 서포트 모듈(필드버스)
  • 안전: E-Stop, 인터록, 과하중/편하중 트립

인터페이스 규격은 "도킹 방식, 통신 프로토콜, 이재 방법"을 조기에 합의해 전체 시스템(AMR·입출고·디지털트윈)의 병렬 개발이 가능하도록 합니다.

11. 다음 단계 (의사결정 체크리스트)

바로 회의용

결정해야 할 5가지

  1. 대상 부재 범위: "절단소재" vs "완성 소조블록" 우선순위
  2. 자동화 강도: 컨셉 A(추천+수동) → B(전동 서포트) → C(셀 기반)
  3. 도킹 방식: 기계 가이드 중심 vs 비전/LiDAR 중심
  4. 안전 기준: 허용 변형/지지 조건(현장 표준) 확정
  5. 성능지표: 공간효율, 적치/피킹 시간, 재배치 비용, 안전 이벤트

추천 결론(초안)

현장성과 + 세계 최초급 스토리를 동시에 잡으려면
컨셉 A로 빠르게 실증 → 컨셉 B로 차별화 → 컨셉 C는 PoC/특허 트랙으로 병렬 추진이 가장 안전합니다.

"랙 구조" 자체보다, 결국 차별화는 형상/CoG 기반 지지점 결정 + 안전 검증 + 생산계획 연계에서 나옵니다.
