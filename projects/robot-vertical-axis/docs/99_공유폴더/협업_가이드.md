# 협업 가이드라인

## 🤝 협업 원칙

### 1. 커뮤니케이션
- **즉시 공유**: 중요한 발견사항은 즉시 팀에 공유
- **명확한 표현**: 모호한 표현 지양, 구체적이고 명확하게
- **정기 업데이트**: 일일 진행상황 필수 업데이트

### 2. 파일 관리
- **파일명 규칙**: `YYMMDD_작성자_제목.확장자`
  - 예: `240917_김철수_상축분석결과.docx`
- **버전 관리**: 수정 시 이전 버전 백업
- **임시 파일**: 99_공유폴더에서 작업 후 해당 폴더로 이동

### 3. 작업 분담
```
작업자1: 문제점 분석 + 기술 검토
작업자2: 대책 수립 + 일정 관리
작업자3: 실행 계획 + 문서 정리
```

## 🚨 긴급 상황 대응

### 즉시 보고해야 하는 상황
- 기존 계획으로 해결 불가능한 기술적 문제 발견
- 일정 지연이 2일 이상 예상되는 경우
- 추가 리소스(인력/예산/장비) 필요한 경우
- 안전상 위험 요소 발견

### 보고 방법
1. **즉시**: 팀 채널에 알림
2. **상세**: 이슈 내용을 문서로 정리
3. **대안**: 가능한 해결책 함께 제시

## 📁 폴더별 사용법

### 01_문제점분석
- 개인별 분석 결과를 각자 업로드
- 템플릿 활용하여 표준화된 형식 유지
- 데이터/이미지는 하위 폴더에 별도 정리

### 02_대책수립
- 여러 대안을 각각 파일로 작성
- 평가 기준에 따른 점수 부여
- 최종 선택 방안에 대한 상세 계획 수립

### 03_일정관리
- 매주 금요일 일정표 업데이트
- 지연 사항 발생 시 즉시 반영
- 리스크 요소 지속적 모니터링

### 04_작업진행
- 매일 개인별 진행상황 입력
- 이슈 발생 시 즉시 기록
- 해결 방법 및 소요시간 문서화

### 05_문서자료
- 회의록은 회의 당일 작성 완료
- 기술 자료는 출처 명확히 기재
- 중요 문서는 백업 필수

### 99_공유폴더
- 임시 작업 공간으로 활용
- 1주일 이상 방치된 파일은 정리
- 팀 공통 의견 수렴 장소

## 💡 효율적 협업을 위한 팁

1. **중복 작업 방지**: 작업 시작 전 팀원들과 확인
2. **표준 템플릿 활용**: 제공된 템플릿 최대한 활용
3. **정기적 동기화**: 매일 오전 간단한 스탠드업 미팅
4. **문서화 습관**: 모든 결정사항과 변경사항 기록
